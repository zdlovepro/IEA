
# 本项目当前落地建议（基于仓库现状）

为保持规约与实际情况一致，建议优先执行以下收敛动作：

1. **目录命名统一**：`frontend/src/constans` → `frontend/src/constants`，避免拼写长期扩散。
2. **Python 服务分层**：`python-service/app` 逐步补齐 `api/`、`schemas/`、`clients/`，避免把所有逻辑堆在 `main.py` 或单层 `services`。
3. **Java 后端结构**：明确 `controller` - `service` - `client` - `mapper` 的标准结构，并统一返回体（`Result`）、全局异常处理器和日志格式。
4. **状态枚举收敛**：课件、会话、问答三类核心状态枚举在文档和代码中统一定义（例如使用 Java `enum`，Python `StrEnum`，前端 `const` 映射），避免前后端各自解释。
5. **接口文档同步**：每次新增接口时，同步更新 `docs/api.md` 和本规约中涉及的约束，不让文档落后于实现。
