

# 软件需求规格说明书 (SRS)
**项目名称：** 基于泛雅平台的AI互动智课生成与实时问答系统
**命题方向：** 智能计算 (应用类)
**企业合作方：** 超星集团

## 1. 引言

### 1.1 编写目的
本文档旨在明确“基于泛雅平台的AI互动智课生成与实时问答系统”的功能需求、非功能需求、接口要求及运行环境等，为项目的设计、开发、测试以及后续与超星泛雅网络教学平台的集成提供基准。

### 1.2 项目背景
在教育数字化转型的浪潮下，个性化学习与高效知识传递成为核心需求。当前泛雅平台虽积累了海量PPT、PDF等优质教学资源，但传统静态课件存在“讲授模式固化、互动性弱”的痛点。本项目旨在依托泛雅平台，引入AI大模型技术，通过“智课生成 + 实时问答”打通“课件解析-内容重构-智能讲授-实时问答-进度续接”五个核心环节，将静态资源转化为具备交互能力的“智能课程”。

### 1.3 目标用户
*   **教师（内容创作者）：** 快速将已有课件转化为互动智课，并能编辑校验AI生成的讲授脚本。
*   **学生（通识课自主学习者）：** 获得沉浸式、可互动的视听学习体验，随时提问并获得精准解答。

## 2. 总体描述

### 2.1 运行环境
*   **客户端支持：** 泛雅平台Web端、学习通移动端。
*   **浏览器要求：** 支持 Chrome 内核及 IE9 以上版本浏览器。
*   **客户端设备要求：** 兼容常规设备（i5 x86处理器／骁龙855及以上，4GB内存，64GB存储）。
*   **网络环境要求：** 上行带宽 ≥ 2 Mbps，下行带宽 ≥ 5 Mbps，延迟 ≤ 100ms，丢包率 ≤ 1%。

### 2.2 服务端及资源要求
*   **硬件配置：** 8核CPU、16GB内存、500GB存储。
*   **并发能力：** 支持并发访问人数 ≥ 10人。

## 3. 核心功能需求 (Functional Requirements)

系统主要由以下三大核心模块构成：

### 3.1 智课生成模块 (Smart Course Generation)
*   **F1.1 课件解析：** 系统需支持对上传的PPT、PDF格式课件进行智能解析，精准提取图表、公式、核心知识点等要素。
*   **F1.2 脚本重构与生成：** 基于提取的知识点，自动生成包含“开场白、核心讲解、过渡语”的结构化讲授脚本。
*   **F1.3 语音合成/数字人对接：** 将生成的脚本对接语音合成API或数字人系统，实现课件内容的“可听”、“可视”化讲授。
*   **F1.4 教师干预与编辑：** 提供后台编辑界面，允许教师对AI生成的结构化脚本进行人工修改、校验与调优。

### 3.2 实时问答交互模块 (Real-time Q&A Interaction)
*   **F2.1 多模态输入介入：** 允许学生在听课过程中随时通过**文字**或**语音**形式打断并进行提问。
*   **F2.2 上下文关联解答：** 问答系统需结合当前讲授的课件上下文（当前页知识点）进行精准解答，确保答案与课件高度相关。
*   **F2.3 多轮交互支持：** 支持针对某一知识点的连续多轮追问与交互答疑。

### 3.3 进度续接与节奏调整模块 (Progress & Pace Management)
*   **F3.1 智能断点续接：** 在学生完成打断提问及答疑后，系统需自动精准定位回原讲授节点，无缝继续播放。
*   **F3.2 学习状态诊断与节奏调整：** 通过NLP技术分析学生提问内容，评估其对知识点的理解程度；针对未理解内容自动补充讲解，并动态调整讲授节奏。
*   **F3.3 进度可视化：** 为学生和教师提供直观的学习进度与知识点掌握情况的可视化面板。

## 4. 非功能性需求 (Non-Functional Requirements)

### 4.1 性能指标
*   **课件解析效率：** 课件解析响应时间 ≤ 2分钟/份。
*   **问答交互效率：** 实时问答响应时间 ≤ 5秒。
*   **算法准确率：** 
    *   知识点识别准确率 ≥ 80%。
    *   智能解答准确率 ≥ 85%。

### 4.2 数据要求
*   系统须支持处理涵盖文、理、工、医等多学科的课件。
*   用于训练/验证的课件解析样本集规模不少于 100 份（需涵盖文、理、工三大类）。

### 4.3 接口与集成要求 (API Requirements)
*   **API提供：** 需提供不少于10个标准化功能接口（例如：课件解析接口、讲授启动接口、问答交互接口等）。
*   **集成方式：** 接口需支持与泛雅平台进行异步调用及轻量化集成。
*   **规范标准：** 需基于超星开放API设计规范自主制定相关的标准化接口。
