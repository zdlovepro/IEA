
# 06-数据库与存储设计规约

## 1. 命名规范

### 1.1 表与字段

- 表名、字段名统一小写下划线。
- 主键统一为 `id`。
- 业务唯一标识使用独立字段，如 `courseware_id`、`session_id`。
- 审计字段统一：`created_at`、`updated_at`。

### 1.2 示例建表语句

```sql
CREATE TABLE `coursewares` (
  `id` BIGINT UNSIGNED NOT NULL AUTO_INCREMENT,
  `courseware_id` CHAR(32) NOT NULL COMMENT '业务唯一ID',
  `name` VARCHAR(255) NOT NULL,
  `status` VARCHAR(32) NOT NULL DEFAULT 'UPLOADED',
  `created_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  `updated_at` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_courseware_id` (`courseware_id`),
  KEY `idx_status_created` (`status`, `created_at`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

## 2. 设计要求

- 核心业务表必须有状态字段，便于表达流程进度。
- 需要追踪异步任务时，单独建设任务表，不把任务状态散落在多个业务表里。
- JSON 字段只存扩展信息，不替代正常关系建模。

## 3. 索引要求

- 高频查询条件、关联字段、唯一约束字段必须建索引。
- 涉及排序、分页、状态筛选的查询要评估联合索引。

## 4. 文件与对象存储

### 4.1 路径规范

格式：`{环境}/{业务模块}/{日期}/{业务ID}/{文件名}`

示例：

```text
# 原始课件
prod/courseware/2026-04-16/cware_abc123/original.pptx

# TTS 生成的音频切片
prod/audio/2026-04-16/cware_abc123/page_001.mp3
```

### 4.2 管理要求

- 原始文件、切片文件、音视频产物要分桶或分目录存储。
- 上传后返回的文件 URL、对象 Key、展示名分开管理，不混作一个字段。
