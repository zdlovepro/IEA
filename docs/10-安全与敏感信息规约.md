# 10-安全与敏感信息规约

## 1. 敏感信息处理

### 1.1 禁止提交到仓库的内容
- API Key、Access Token、Secret Key
- 数据库密码、Redis 密码、消息队列密码
- 对象存储（MinIO/S3/OSS）的 AccessKey/SecretKey
- 第三方服务（大模型、TTS、数字人）的 Token 或签名密钥
- 私钥证书、JWT 签名密钥
- 包含生产环境 IP、端口、域名的未脱敏配置

### 1.2 配置管理要求
- 所有敏感信息必须通过环境变量或密钥管理服务（如 HashiCorp Vault、K8s Secret）注入。
- 本地开发可使用 `.env` 文件，但必须将 `.env` 加入 `.gitignore`。
- 提供 `.env.example` 模板，说明需要哪些环境变量，但不包含真实值。
- 生产配置与代码仓库严格分离，禁止在 `application-prod.yml` 中硬编码密钥。

### 1.3 日志脱敏要求
- 日志中禁止打印：
  - 完整的密码（即使是用户输入的）
  - JWT Token 全文（可打印前几位或后几位用于调试）
  - 身份证号、手机号（如需打印，中间4位脱敏，如 `138****1234`）
  - 数据库连接串中的密码部分
- 对于不确定是否包含敏感信息的对象，在打印前调用脱敏工具类。

### 1.4 示例：日志脱敏工具

```java
public class SensitiveDataUtil {
    public static String maskPhone(String phone) {
        if (phone == null || phone.length() < 7) return "***";
        return phone.replaceAll("(\\d{3})\\d{4}(\\d{4})", "$1****$2");
    }
    
    public static String maskToken(String token) {
        if (token == null || token.length() <= 8) return "***";
        return token.substring(0, 4) + "****" + token.substring(token.length() - 4);
    }
}
```

## 2. 接口安全

### 2.1 认证与授权
- 涉及用户身份、资源操作的接口必须经过认证（除公开接口如健康检查外）。
- 使用 JWT 或 Session 机制，Token 有效期合理设置（建议 Access Token 2小时，Refresh Token 7天）。
- 接口需校验当前登录用户是否有权访问目标资源（如课件归属、会话归属）。

### 2.2 输入校验
- 所有来自前端的参数必须校验：
  - 类型、长度、范围、格式（使用注解如 `@NotNull`, `@Size`, `@Pattern`）。
  - 文件上传：限制文件类型（MIME 白名单）、大小（如 ≤100MB）、文件名不能包含路径穿越字符（`../`）。
- SQL 注入防护：使用 MyBatis 的 `#{}` 参数化查询，禁止拼接用户输入到 SQL 中。
- XSS 防护：前端渲染时对用户输入转义，后端也可对输出做过滤。

### 2.3 限流与防滥用
- 对外接口（尤其是上传、问答、短信验证码）建议配置限流：
  - 基于用户 ID 或 IP 的 QPS 限制。
  - 大文件上传限制并发数。
- 验证码：敏感操作（登录、修改密码）应使用验证码或二次确认。

### 2.4 敏感接口额外保护
- 涉及支付、权限变更、批量操作的接口需记录操作日志并建议使用二次校验（如密码确认）。

## 3. 第三方调用安全

### 3.1 调用外部大模型、TTS、数字人服务
- API Key 和 Secret 必须从环境变量读取，不得硬编码。
- 请求参数中如果包含用户输入，需进行清洗（去除恶意指令）。
- 第三方返回的内容若直接展示给用户，应做内容安全检测（如过滤色情、暴力等）。

### 3.2 回调接口验证
- 如果 Python 服务或第三方通过回调通知结果，必须验证回调请求的合法性（如签名校验、IP 白名单）。
- 避免 SSRF（服务端请求伪造）：调用外部 URL 时限制目标地址为内部白名单或禁止访问内网 IP。

### 3.3 对象存储安全
- 临时上传 URL 应设置有效期（如 5 分钟）。
- 公开访问的媒体资源若涉及隐私，使用带签名的 URL 或临时凭证。
- 敏感文件（如包含学生信息的解析结果）应设置为私有读，通过后端代理访问。

## 4. 数据传输安全

- 生产环境必须启用 HTTPS（TLS 1.2 或以上）。
- 内部服务间调用（Java ↔ Python）如果在公网传输，也应使用 HTTPS 或 mTLS。
- 禁止明文传输密码或 Token，必须放在 `Authorization` 头或加密后的 Body 中。

## 5. 开发与测试环境安全

- 测试环境可使用弱密码或默认密钥，但严禁与生产环境共用任何密钥或数据库。
- 测试数据应脱敏，避免使用真实用户信息。
- 本地调试时不得绕过认证机制（除非使用 Mock 账号）。

## 6. 常见风险点检查清单（Code Review 时使用）

- [ ] 是否在代码中硬编码了任何密钥或密码？
- [ ] 日志中是否可能打印敏感信息（Token、密码、用户手机号）？
- [ ] 文件上传是否限制了类型和大小，且文件名是否安全？
- [ ] 是否对用户输入进行了校验（非空、格式、注入防护）？
- [ ] 关键接口是否实现了认证和权限校验？
- [ ] 对外部服务调用是否设置了超时和重试，避免挂起？
- [ ] 是否使用了 HTTPS 传输敏感数据？
- [ ] 对象存储的临时 URL 是否设置了有效期？

## 7. 违规处理

- 代码合并前若发现硬编码密钥，Reviewer 必须打回并要求立即修改。
- 因敏感信息泄露导致安全事件，按照项目安全应急预案处理。
